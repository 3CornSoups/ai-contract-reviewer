# 后端模块说明 (FastAPI)

## 1. 模块职责
后端使用 FastAPI 框架，作为系统的业务控制中枢，负责：
- 接收并存储 PDF 文件（包括法律法规与待审合同）。
- 解析 PDF 并切分文本段落 (Chunking)。
- 调用 OpenAI API 生成文本 Embedding，将其存入 pgvector 数据库。
- 合同上传后，进行 RAG（检索增强）相似度匹配，组装 Prompt 提交给 LLM 审核。
- 封装成 RESTful API 供前端与 Nginx 网关调用。

## 3. 依赖关系
- `FastAPI` + `Uvicorn`: 提供高并发异步Web服务。
- `SQLAlchemy` + `pgvector`: ORM框架与向量存储对接，简化SQL拼写并支持 `Vector` 数据类型映射。
- `openai (v0.28.1)`: 用于对接兼容 OpenAI API 规范的大模型（通过修改 `api_base` 支持本地/三方大模型）。
- `pdfplumber` / `PyPDF2`: PDF文件文本提取。

## 4. 核心逻辑与修改风险点
- **PDF 分块逻辑 (Chunking)**：`utils/pdf_parser.py` 中的分块大小直接影响 RAG 召回的法条完整度。当前固定字数+重叠字符的切分逻辑较简单，后续可升级为按“法条标题”语义切分。
- **Embedding 维度约束**：`models/schemas.py` 中 `Vector(1536)` 是硬编码的，如果更换了非 OpenAI 接口产生其他维度的向量（如 `768`），**必须**同步修改数据库建表语句和 ORM 类的维度参数，并重建表。
- **并发与限流**：FastAPI 内置仅支持异步IO并发处理，真正的请求频率限流依赖外层 Nginx 的 `limit_req` 配置。
