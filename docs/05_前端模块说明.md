# 前端模块说明 (Vue)

## 1. 模块职责
前端基于 Vue 3 + Vite 构建，结合 Element-Plus 组件库，主要负责：
- 提供清晰直观的交互界面。
- 上传法律文件以进行知识库构建。
- 上传合同并展示由大模型(LLM)生成的审核报告（包括总览、违规点、法律依据及建议）。
- 处理前后端 API 通信及异常提示。

## 2. 文件结构
```text
frontend/
├── index.html           # 页面入口
├── package.json         # 项目依赖
├── vite.config.js       # Vite配置及开发环境代理设置
├── src/
│   ├── main.js          # Vue实例初始化、引入全局组件
│   ├── App.vue          # 根组件：包含左侧导航菜单与路由页面容器
│   ├── api/
│   │   └── index.js     # Axios API 请求封装
│   └── views/
│       ├── AuditView.vue    # 合同审核与结果展示视图
│       └── KnowledgeView.vue# 知识库PDF上传视图
```

## 3. 依赖关系
- `vue` (v3.x): 核心框架，使用 Composition API (`<script setup>`)。
- `vite`: 极速构建工具，取代传统的 Webpack。
- `element-plus`: UI 组件库，用于布局、菜单、上传组件、手风琴展示等。
- `axios`: 发送 HTTP 请求，在拦截器中统一处理了错误信息（如后端验证报错、限流拦截等）。

## 4. 核心逻辑与组件
### 4.1 页面切换
未采用 `vue-router`，而是基于 `App.vue` 中的 `<component :is="...">` 实现了简单的单页组件切换逻辑，降低了项目复杂度。
- `AuditView`：审核视图
- `KnowledgeView`：知识库视图

### 4.2 API 通信
`src/api/index.js` 使用 axios 封装了对外的 API 调用：
- 关键点：文件上传需要通过 `FormData` 组装，并设置 header `Content-Type: multipart/form-data`。
- 因为 LLM 生成响应可能耗时较长，axios 实例的超时时间（timeout）设置为 `120000` (2分钟)。

## 5. 修改风险点
- **代理配置**: `vite.config.js` 中的 proxy 设置仅在开发模式(`npm run dev`)有效。在生产部署时，前端发出的请求会带上 `/api` 前缀，依赖 Nginx 将其反向代理至后端 `8000` 端口。请勿在 axios baseURL 中写死 `http://localhost:8000`。
- **UI展示限制**: 当前展示的 JSON 结果依赖后端的严格结构约束 (`status`, `risk_level`, `violations` 等)，如果后端的 Prompt 被修改导致大模型返回的字段名变化，前端渲染将会报错或留白，请确保前后端字段一致。
