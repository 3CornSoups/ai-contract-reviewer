# 部署文档

## 1. 概述
本项目采用 Docker 容器化编排方案，所有的服务（前端、后端、网关、数据库）都可以通过一个 `docker-compose.yml` 文件一键拉起。无需在宿主机上手动安装 Python 环境、Node 环境或 PostgreSQL。

## 2. 环境准备
确保目标部署服务器（Linux/Mac/Windows）已安装：
- **Docker**: (>= 20.10)
- **Docker Compose**: (>= 1.29)
- **外部网络**: 服务器能够正常访问所配置的 `OPENAI_API_BASE` 地址。

## 3. 部署步骤

### 3.1 环境变量配置
为了保证安全性，不要在代码中硬编码大模型的 API Key。
在项目根目录（`docker-compose.yml` 所在位置）创建一个 `.env` 文件（不要提交到 Git）：
```env
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
# 如果使用第三方转发或本地模型(如 Ollama)，请修改以下配置：
OPENAI_API_BASE=https://api.openai.com/v1
```

### 3.2 启动服务
在项目根目录执行以下命令，该过程会自动构建镜像并启动所有容器：
```bash
docker-compose up -d --build
```
或使用项目提供的脚本（需赋予执行权限 `chmod +x scripts/*.sh`）：
```bash
./scripts/start.sh
```
参数说明：
- `-d`：后台运行容器。
- `--build`：强制重新构建镜像，确保代码最新。

### 3.3 验证服务状态
执行命令查看容器状态：
```bash
docker-compose ps
```
应看到 `reviewer_db`、`reviewer_backend`、`reviewer_gateway` 三个容器的状态均为 `Up`。

此时，打开浏览器访问 `http://<服务器IP>` 即可进入系统前端页面。

### 3.4 停止与清理
停止并移除容器（保留数据卷）：
```bash
docker-compose down
```
或执行 `./scripts/stop.sh`。若想清理掉数据库数据（注意：此操作不可逆！）：
```bash
docker-compose down -v
```

## 4. 常见问题排查

**Q1：页面能够打开，但上传文件或审核时一直转圈或报错。**
- **排查**：使用 `docker logs reviewer_backend` 查看后端日志。通常是因为 `OPENAI_API_KEY` 无效或所在服务器无法连通外部 API。

**Q2：提示数据库连接失败。**
- **排查**：数据库容器初次启动初始化需要一定时间。请等待几秒钟再试。检查 `reviewer_db` 日志 `docker logs reviewer_db`，确保出现 `database system is ready to accept connections`。

**Q3：前端上传大于 1MB 的 PDF 报错。**
- **排查**：前端请求经由 Nginx 转发，已在 `nginx/nginx.conf` 中配置 `client_max_body_size 20M;`。若依然报错，请检查配置文件是否正确挂载到了容器内。
