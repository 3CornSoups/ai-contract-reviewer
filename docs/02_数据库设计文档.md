# 数据库设计文档

## 1. 概述
本项目采用 PostgreSQL 作为核心数据库，并强依赖 `pgvector` 插件实现高维向量的存储与相似度检索。业务关系型数据与向量数据统一存储在同一个实例中，极大降低了系统运维复杂度。

## 2. 表结构说明

### 2.1 用户表 (`users`)
用于系统基础登录鉴权。
| 字段名 | 类型 | 约束 | 备注 |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | 主键自增 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名 |
| password_hash | VARCHAR(255)| NOT NULL | 密码哈希 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 2.2 知识库文档表 (`knowledge_docs`)
记录已上传的法律法规文档的元信息。
| 字段名 | 类型 | 约束 | 备注 |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | 主键自增 |
| filename | VARCHAR(255) | NOT NULL | 原始文件名 |
| file_path | TEXT | NOT NULL | 在服务器或对象存储上的路径 |
| upload_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 上传时间 |

### 2.3 向量分块表 (`knowledge_chunks`) (核心表)
存储用于 RAG 检索的文本片段及对应的 Embedding 向量。
| 字段名 | 类型 | 约束 | 备注 |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | 主键自增 |
| doc_id | INTEGER | FOREIGN KEY REFERENCES knowledge_docs(id) ON DELETE CASCADE | 关联的文档ID |
| content | TEXT | NOT NULL | 切分后的文本片段 |
| embedding | vector(1536) | | 文本向量（维度视具体模型而定，此处以OpenAI 1536维为例） |

### 2.4 审核记录表 (`audit_records`)
存储 LLM 审核生成的结构化报告。
| 字段名 | 类型 | 约束 | 备注 |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | 主键自增 |
| contract_filename| VARCHAR(255) | NOT NULL | 被审核的合同文件名 |
| audit_result | JSONB | NOT NULL | 包含风险点、建议的 JSON 格式数据 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 审核完成时间 |

### 2.5 操作日志表 (`operation_logs`)
记录关键操作（知识库上传、合同审核等），便于审计与问题排查。
| 字段名 | 类型 | 约束 | 备注 |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | 主键自增 |
| operation_type | VARCHAR(50) | NOT NULL | 操作类型，如 knowledge_upload、contract_audit |
| detail | TEXT | | 可选说明（如文件名、doc_id） |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 操作时间 |

## 3. 索引设计与向量检索逻辑
- **HNSW 索引**：为 `knowledge_chunks` 表的 `embedding` 字段建立了 HNSW (Hierarchical Navigable Small World) 索引 `CREATE INDEX ON knowledge_chunks USING hnsw (embedding vector_cosine_ops);`。
- **检索逻辑**：使用余弦相似度操作符 `<=>` 进行最邻近检索。例如：`SELECT content FROM knowledge_chunks ORDER BY embedding <=> '[...向量值...]' LIMIT 5;` 即可召回最相似的 5 个法条分块。

## 4. 修改风险点
- `embedding` 字段的维度（当前 1536）必须与后端调用的 Embedding 模型输出的维度严格一致。若更换为开源小模型（如 768 维），必须修改建表 SQL 和重新初始化。
